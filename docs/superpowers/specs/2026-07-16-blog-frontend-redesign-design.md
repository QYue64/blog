# 雀雀博客前端重设计规格

## 1. 背景

当前项目是基于 Nuxt 3、Nuxt Content 和 Alpine 主题二次开发的静态博客。根目录提供本地主题源码，`.starters/default` 是实际运行的博客应用。现有内容、Markdown 路由和 GitHub Pages 发布流程已经可用，本次只重构前端主题层。

用户选择的视觉方向为“数字标本室”：将网站设计成一套属于雀雀的个人网络操作系统，使用浅色背景、超大字体、等宽元数据、漂浮窗口、有机色块和克制动效形成辨识度。

## 2. 目标

- 让访客在首页前 5 秒内记住网站的独特视觉风格。
- 重做首页、文章列表、文章详情、导航和页脚。
- 同时使用字体、非常规布局、图形纹理和动效，但保证文章易读。
- 保留全部文章正文、文章 URL、Nuxt Content 数据源和静态发布流程。
- 桌面端和移动端都达到完整、可用的设计质量。

## 3. 非目标

- 不新增搜索、标签筛选、独立归档页或独立关于页。
- 不修改文章正文内容和现有文章 URL。
- 不引入服务端接口、数据库或用户账户。
- 不提供深色主题。
- 不依赖文章封面完成核心布局。

## 4. 视觉系统

### 4.1 色彩

| 用途 | 色值 |
| --- | --- |
| 页面背景 Ice | `#EDF4FF` |
| 主要文字 Ink | `#172030` |
| 主图形 Lime | `#C9FF5C` |
| 交互强调 Violet | `#7656EF` |
| 辅助窗口 Pink | `#FFB8DB` |
| 白色浮层 | `rgba(255, 255, 255, 0.87)` |
| 次要文字 | `#596578` |
| 次要边框 | `#9BA7BA` |

页面始终使用浅色背景。酸性绿负责大型有机图形，紫色负责链接、进度和当前状态，粉色只用于少量窗口或悬停反馈。

### 4.2 字体

- 大标题：粗体无衬线字体栈，中文优先使用 `PingFang SC`，英文使用系统 Grotesk/Arial 风格。
- 强调词：衬线斜体字体栈，用于 `homepage.` 等少量品牌词。
- 导航、元数据、编号和代码：系统等宽字体栈。
- 正文：系统无衬线中文字体栈，桌面端正文宽度不超过约 `68ch`，行高约 `1.8`。

不新增远程字体依赖，避免字体服务在国内网络环境下失效。

### 4.3 图形与容器

- 主图形使用不规则有机色块，而不是通用渐变背景。
- “窗口”组件使用 1px 墨色边框和 6–8px 硬阴影。
- 页面主体不使用重复的圆角卡片网格。
- 文章列表采用日志表格/开放列表；正文采用稳定的开放式阅读区域。
- 圆角仅用于浏览器式容器、移动菜单和必要的触控容器。

### 4.4 动效

- 首页有机色块缓慢漂移。
- `NOW.TXT` 和 `STATUS` 窗口产生轻微视差或浮动。
- 关键词带横向循环移动。
- 文章列表悬停时才显示文章预览窗口。
- 页面切换和菜单使用 180–260ms 的轻量过渡。
- `prefers-reduced-motion: reduce` 时关闭循环动画、视差和非必要位移。
- 正文滚动过程中不移动正文列，不使用会影响阅读定位的动画。

## 5. 信息架构

### 5.1 顶部导航

- 品牌：`QYUE://HOME`，点击返回首页。
- `文章`：进入现有 `/articles`。
- `关于`：滚动或链接到首页的 `#about` 区域，不新增页面。
- `链接`：滚动或链接到首页的 `#links` 区域，不新增页面。
- `菜单`：打开移动端/窄屏导航面板；桌面端作为外部链接入口。

### 5.2 外部链接

现有“笔记、AI、DSM、Terminal”保留，集中放入首页 `#links`、菜单和页脚，避免挤占首屏主导航。

## 6. 首页

### 6.1 首屏

允许出现的主要文案如下：

- 品牌：`QYUE://HOME`
- 标题：`好奇心，有自己的 homepage.`
- 元信息：`AI · HOMELAB · DEVOPS`
- `NOW.TXT`：`正在研究如何让 AI 工具真正融入日常工作流。`
- `STATUS`：`BUILDING / DOCUMENTING / SHARING`

首屏使用一个右上方酸性绿有机色块和两个漂浮窗口。窗口不能遮挡主标题、导航或后续主要操作。

### 6.2 关键词带

首屏下方放置横向移动的关键词带，内容来自博客真实主题：`AI · HOMELAB · DOCKER · OPENCLAW · CLOUDFLARE · NOTES`。它只承担视觉节奏，不作为筛选控件。

### 6.3 最新文章

- 查询 `articles` 下按日期倒序的第一篇文章。
- 显示文章编号、标题、描述、日期、预计阅读时间和入口。
- 不依赖封面图；视觉重点来自编号、排版和颜色。

### 6.4 最近记录

- 展示最新三篇文章。
- 每行显示编号、标题、分类和年月。
- 整行可点击，键盘聚焦时与悬停时提供等效反馈。

### 6.5 关于与链接

- `#about` 使用短段落说明网站内容方向，不新增复杂个人履历。
- `#links` 呈现笔记、AI、DSM、Terminal 外部入口。
- 页脚显示版权、备案入口和必要外部链接。

### 6.6 移动端

- 首屏标题缩小但保持三行以内。
- 漂浮窗口移动到标题下方，不覆盖正文。
- 最近文章表格隐藏分类列，只保留编号、标题和日期。
- 导航折叠成菜单，菜单可通过按钮、Esc 和点击遮罩关闭。

## 7. 文章索引页

- 页面品牌切换为 `QYUE://LOGS`。
- 主标题为 `所有记录。`。
- 顶部显示 `NEWEST_FIRST` 和总文章数，不提供无实际功能的筛选控件。
- 每行显示编号、标题、分类和完整日期。
- 桌面端悬停或键盘聚焦时，在行附近显示预览窗口。
- 移动端不显示预览窗口和分类列，整行保持至少 44px 触控高度。
- 列表继续使用 `queryContent('articles').sort({ date: -1 })` 获取数据。

## 8. 文章阅读页

### 8.1 页头

- 顶部显示阅读进度条。
- 品牌格式为 `QYUE://ENTRY_XXX`。
- 显示文章编号、日期、预计阅读时间、标题和元数据窗口。
- 页头可以使用一个酸性绿色块，但不能延伸到正文列下方。

### 8.2 桌面布局

正文区域分为三列：

1. 左侧：文章编号和返回文章列表。
2. 中间：最大宽度约 `68ch` 的正文。
3. 右侧：根据正文标题生成的目录。

正文内容区域不使用漂浮窗口、视差或循环动画。

### 8.3 移动布局

- 使用单列正文。
- 隐藏右侧目录，保留阅读进度。
- 返回文章列表放在文章标题与正文之间。
- 代码块允许横向滚动，不允许撑破页面。

### 8.4 阅读增强

- 阅读进度根据文章滚动范围更新。
- 目录从 Nuxt Content 提供的 TOC 数据生成，并高亮当前章节。
- 代码块使用浅色背景、粉色标题栏和复制按钮。
- 文章底部显示上一篇和下一篇；不存在时明确显示无更多文章，不生成无效链接。
- 外部链接保留明显的外链提示和安全属性。

## 9. 组件边界

建议新增或重构以下组件：

| 组件 | 职责 |
| --- | --- |
| `AppHeader` | 桌面导航、品牌入口和菜单触发器 |
| `AppMobileMenu` | 移动导航、外部链接和焦点管理 |
| `SpecimenWindow` | 统一窗口边框、标题栏和阴影，不负责业务数据 |
| `HomeSpecimenHero` | 首页标题、有机图形和状态窗口 |
| `KeywordTicker` | 关键词带及减少动态效果降级 |
| `ArticleLogList` | 文章索引和首页最近记录的容器 |
| `ArticleLogRow` | 单条文章日志、悬停预览和键盘状态 |
| `ArticleHeader` | 文章编号、标题、日期和元数据窗口 |
| `ReadingProgress` | 文章滚动进度 |
| `ArticleToc` | 目录和当前章节状态 |
| `ArticlePagination` | 上一篇和下一篇 |
| `ProsePre` | 浅色代码块和复制交互 |

现有 `ArticlesList` MDC 调用保持兼容，可以内部改为组合新的日志列表组件，避免修改 `2.articles.md` 的使用方式。

## 10. 数据流与元数据

1. Nuxt Content 继续从 `.starters/default/content` 加载 Markdown。
2. 首页和文章索引通过同一个查询工具获取文章，先按日期倒序、再按 `_id` 倒序处理同日内容；缺少日期的文章排在最后。
3. 文章编号优先解析现有文件名的数字前缀（`01-` 至 `10-`），保证新增文章后旧编号不变化；无法解析时才使用排序位置作为降级编号。
4. 为现有十篇文章补充可选 `category` frontmatter；正文和 URL 不变。
5. 未提供 `category` 时显示 `NOTE`。
6. 阅读时间通过本地工具函数遍历 Nuxt Content AST 文本估算：中文约 400 字/分钟，英文约 220 词/分钟，最少显示 1 分钟。
7. 当前文章页通过同一个排序工具查找上一篇和下一篇，避免首页、索引和文章页采用不同排序规则。

## 11. 缺失数据与异常处理

- 缺少描述：隐藏摘要，不显示空白占位。
- 缺少分类：显示 `NOTE`。
- 缺少日期：显示 `UNDATED`，并排在有日期内容之后。
- 没有正文标题：隐藏目录列，正文保持居中。
- 没有上一篇或下一篇：显示不可点击的“暂无更多文章”。
- 外部图片加载失败：不影响首页、索引和文章页核心结构。
- 代码复制失败：保留可选择的代码文本，不阻断阅读。
- JavaScript 不可用：文章、导航链接和主要内容仍可阅读；动效和复制功能自然降级。

## 12. 可访问性

- 所有交互元素支持键盘操作和明显焦点状态。
- 菜单打开后管理焦点，并支持 Esc 关闭。
- 装饰色块和窗口标记为非语义内容。
- 文字与背景对比度达到 WCAG AA。
- 触控目标至少 44×44px。
- 页面语言改为 `zh-CN`，日期改为中文格式。
- 动效遵循 `prefers-reduced-motion`。

## 13. 验收与测试

### 13.1 自动检查

- `pnpm lint`
- `pnpm generate`
- 静态生成后检查首页、`/articles` 和全部现有文章路由。

### 13.2 浏览器检查

- 桌面基准：1440×900。
- 移动基准：390×844。
- 额外检查 768px 附近的导航和三列到单列切换。
- 验证菜单打开/关闭、键盘焦点、文章行悬停/聚焦、目录跳转、章节高亮、阅读进度、代码复制和上下篇导航。
- 检查所有页面无横向溢出、内容遮挡或不可读文本。
- 检查减少动态效果模式下没有循环移动或视差。

### 13.3 视觉验收

- 首页首屏的品牌文案、色彩、窗口位置和下一段内容预览与批准的“数字标本室”设计一致。
- 文章索引保持数据日志结构，不退化为卡片网格。
- 阅读页正文列稳定，装饰元素不进入正文。
- 桌面端和移动端分别截图，并与最终概念图逐项比较布局、字体、色彩、间距、窗口样式和响应式行为。

## 14. 完成标准

- 所有现有文章 URL 可访问，正文内容未被改写。
- 首页、文章索引和文章详情形成统一的“数字标本室”视觉系统。
- 移动端菜单、文章列表和正文完整可用。
- lint 和静态生成通过。
- 浏览器验证和概念图对比没有剩余的可修复视觉偏差。
